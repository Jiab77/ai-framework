## ⚙️ Préférences de Co-Développement

* **Rythme de travail :** Réfléchi et délibéré. On valide l'architecture avant de coder.
* **Gestion des désaccords :** Attente forte que l'IA challenge les propositions humaines si une approche plus logique, plus sécurisée ou plus performante existe.
* **Gestion des commentaires :** Code auto-documenté privilégié. Commentaires concis uniquement là où la logique métier est complexe ou non évidente.
* **Revues de code :** Pour les modifications importantes, présenter un résumé des impacts (avant/après) ou le plan d'action détaillé avant d'appliquer les changements.

## 🚫 Préférences Personnelles & Règles d'Or (Pet Peeves)

* **Pas d'emojis** dans les messages de commit ou dans les commentaires du code source (sauf demande explicite).
* **Pas de suppositions** : L'IA doit lire les fichiers réels du projet plutôt que d'imaginer leur contenu ou leur structure.
* **Strict Périmètre** : Ne jamais modifier de fichiers ou de portions de code en dehors du périmètre strict de la tâche en cours, sauf pour corriger une faille de sécurité immédiate.
* **Préservation des patterns** : Préférer l'extension des patterns existants dans le projet plutôt que l'introduction arbitraire de nouvelles bibliothèques ou de nouveaux frameworks d'abstraction.
